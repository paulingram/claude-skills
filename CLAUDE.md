# architect-team plugin

## Codebase Overview

This repo IS the source of the `architect-team` Claude Code plugin — a spec-to-production multi-agent coding pipeline. As of **v1.1.0** it ships **26 skills, 27 named agents, 11 slash commands**, 3 enforcement hooks (wired for both `PostToolUse(TaskUpdate)` / `SubagentStop` and the teams-mode `TaskCompleted` / `TeammateIdle` triggers), cross-platform setup scripts, an opt-in per-project email notifier, the v1.0.0 cross-session lock layer (`hooks/locks.py`), the v1.0.0 teams-mode detection helper (`scripts/setup/teams_mode.py`), and the new v1.1.0 worktree-aware state-resolution helper (`scripts/setup/worktree_paths.py`). **v1.1.0 closes the cross-session coordination gap for worktree-based parallel runs** via a 3-layer model: filesystem isolation via git worktrees (each session edits/tests/commits in its own working tree on its own branch), architectural coordination via the v1.0.0 `.architect-team/locks/` JSON lock layer resolved to the MAIN worktree, and context sharing via the MemPalace store also resolved to the MAIN worktree. Two concurrent `/architect-team` invocations in two worktrees of the same repo now get true filesystem isolation AND see the same lock files + the same MemPalace context — closing the v1.0.0 gap where each worktree's lock dir and MemPalace were isolated, defeating the cross-session coordination intent. (Single-session users see ZERO behavior change.) **v1.0.0 made Claude Code's experimental Agent Teams primitive the default dispatch mode** — long-lived named teammates with their own 1M context windows, a shared task list, direct `SendMessage`, and a Lead that owns coordination. Teams mode requires the `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` env var (or the equivalent in `~/.claude/settings.json`) AND Claude Code ≥ 2.1.32 — without those, the pipeline silently falls back to subagents mode (the v0.10.0 behavior, unchanged). A `--no-teams` flag on any of the three pipeline commands forces subagents mode even when teams are available. Every nested-team pattern in the pipeline (`task-reviewer ×3`, `editability-reviewer ×3`, `interaction-reviewer ×3`, `integration-explorer ×3 + master-synthesizer`, `visual-capture + visual-analyzer`, `diagnostic-researcher ×3`, `codebase-map-reviewer ×3`, `flow-explorer ×3`, `flow-executor ×3`) flattens to Lead-owned dispatch per the Agent Teams docs' "no nested teams" constraint. The plugin orchestrates **MemPalace wake-up → Phase −2 (Triage) → Phase −1 → 8** against a requirements folder, with sibling pipelines for **bug-fix** (`/architect-team:bug-fix`), **UX testing** (`/architect-team:ux-test`, v0.9.29), and **mini** (`/architect-team:mini`, v0.10.0).

**Stack:** Markdown (skills/agents/commands), JSON (plugin/marketplace/hooks metadata), Python 3.10+ (hooks + setup scripts + the v1.0.0 `scripts/setup/teams_mode.py` detector + the v1.0.0 `hooks/locks.py` cross-session lock layer + the new v1.1.0 `scripts/setup/worktree_paths.py` worktree-aware state-resolution helper + the stdlib-only `scripts/notify/` email notifier + 1694 pytest self-tests + 1 skipped).

**Structure:** `.claude-plugin/` (identity), `skills/` (26 dirs), `agents/` (27 files), `commands/` (11 files), `hooks/` (JSON wiring + 3 enforcement scripts + a shared `review_evidence_schema.py` module — review-gate evidence schema v6 — + the `locks.py` cross-session lock layer), `scripts/setup/` (`setup.py` + `install_mempalace.py` + `teams_mode.py` detector + the new v1.1.0 `worktree_paths.py` resolution helper), `scripts/notify/` (`notify.py` — the best-effort, stdlib-only per-project email notifier), `tests/` (1694 pytest tests + 1 skipped across 73 test files with structural + cross-consistency coverage), `docs/` (CODEBASE_MAP.md, INTEGRATION_MAP.md, `superpowers/` historical design + plan).

For full architecture, file purposes, conventions, gotchas, and a navigation guide, see [`docs/CODEBASE_MAP.md`](docs/CODEBASE_MAP.md).

## Conventions at a glance

- Author commits with explicit override (repo's local git config has a "Paul Ingrram" typo):
  `git -c user.name="Paul Ingram" -c user.email="paulingram@users.noreply.github.com" commit ...`
- Run the test suite via `python -m pytest -v` from repo root (1694 PASS + 1 SKIPPED expected as of v1.1.0).
- Bump version: update `.claude-plugin/plugin.json` + `.claude-plugin/marketplace.json` + `CHANGELOG.md`.
- Runtime state lives under `.architect-team/` (gitignored). v1.1.0 splits it into **shared** (in main worktree: `locks/`, `.mempalace/`, `run-history/`) and **per-run** (in current worktree: `reviews/`, `teammates/`, `handoffs/`, this-run's `openspec/changes/<slug>/`). The `scripts/setup/worktree_paths.py` helper resolves both via `shared_state_dir()` / `run_state_dir()`.
