# CLAUDE TEAM SIX (CT6)

> Internal plugin slug: `architect-team` (preserved for backward compatibility with existing installations + the 23 slash commands). CLAUDE TEAM SIX is the user-facing display name introduced in v2.2.0.

## What this repo is

This repo IS the source of the **CLAUDE TEAM SIX** Claude Code plugin (internal slug `architect-team`) ‚Äî a spec-to-production multi-agent coding pipeline. It orchestrates **MemPalace wake-up ‚Üí Phase ‚àí2 (Triage) ‚Üí Phase ‚àí1 ‚Üí 8** against a requirements folder, with sibling pipelines for **bug-fix** (`/architect-team:bug-fix`), **UX testing** (`/architect-team:ux-test`), and **mini** (`/architect-team:mini`).

Current shape (v3.43.0): **48 skills** (incl. the phenotype subsystem with 4 seeded phenotypes ‚Äî `user-management`, `config-management`, `ai-management`, `code-wiki`), **39 named agents**, **23 slash commands**, the `services/` server tier (the `librarian` + `triage` + `session-review` + `seeded-mempalace` services + the REPO-1‚Ä¶4 separation manifest; the Librarian is installable via `/architect-team:librarian-install`), the hook-enforced discipline stack (evidence-schema-v7 review gates, run continuity, the instruction-compliance lint wired as an enforced suite gate, the real-time skill-invocation hard-gate), the external-LLM gateway with a selectable secondary provider (OpenAI Codex or Z.ai GLM 5.2), the v3.43.0 delivery-adversarial Opus model split (the ship state: the 12 delivery + adversarial agents on Opus 4.8, the planning / validation / review agents on Fable), the v3.42.0 instruction-surface layer (`docs/ETHOS.md` compiled into every agent + the 5 pipeline skills, the generated `docs/CAPABILITY_INDEX.md`, the `docs/CHANGELOG_RUBRIC.md` gate, the recall data-envelope, and the opt-in behavioral eval tier), and **5901 pytest self-tests + 16 skipped** across 209 test files (measured on macOS without PyYAML ‚Äî 14 PyYAML-gated + 2 archived-coverage-map skips; identical counts under the default env and `PYTHONUTF8=1`).

The default dispatch mode is Claude Code's experimental **Agent Teams** primitive (long-lived named teammates with their own 1M context windows, a shared task list, direct `SendMessage`, and a Lead that owns coordination); it requires `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` and Claude Code ‚â• 2.1.32, and silently falls back to subagents mode otherwise. `--no-teams` forces subagents mode.

## Conventions at a glance

- Author commits with explicit override (repo's local git config has a "Paul Ingrram" typo):
  `git -c user.name="Paul Ingram" -c user.email="paulingram@users.noreply.github.com" commit ...`
- Run the test suite via `python -m pytest -v` from repo root (green under both Windows cp1252 and `PYTHONUTF8=1`; the expected pass/skip count for the current release is in the CHANGELOG entry).
- Bump version: update `.claude-plugin/plugin.json` + `.claude-plugin/marketplace.json` + `CHANGELOG.md`.
- Runtime state lives under `.architect-team/` (gitignored). v1.1.0 splits it into **shared** (in main worktree: `locks/`, `run-history/`) and **per-run** (in current worktree: `reviews/`, `teammates/`, `handoffs/`, this-run's `openspec/changes/<slug>/`); the MemPalace itself lives at the workspace root `.mempalace/palace`, NOT under `.architect-team/`. The `scripts/setup/worktree_paths.py` helper resolves both via `shared_state_dir()` / `run_state_dir()`. **v1.2.0** auto-creates the run worktree (default; `--no-worktree` opts out) via `scripts/setup/worktree_lifecycle.py::create_run_worktree(slug)` ‚Äî branch `architect-team/<slug>`, path `<parent-of-repo>/<repo-name>-<slug>/`. **v1.3.0** auto-cleans merged worktrees on the next run via `cleanup_merged_worktrees()` (with `exclude_current=True` keeping the cwd safe); the `/architect-team:cleanup-worktrees` command is the on-demand variant. **v3.7.0** auto-merges a clean run's branch `--no-ff` into `main` and prunes it (`--no-auto-merge` opts out; a conflict falls back to a PR, never `--force`).
