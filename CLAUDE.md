> Internal plugin slug: `architect-team` (preserved for backward compatibility with existing installations + the 23 slash commands). CLAUDE TEAM SIX is the user-facing display name introduced in v2.2.0.

## What this repo is

This repo IS the source of the **CLAUDE TEAM SIX** Claude Code plugin (internal slug `architect-team`) — a spec-to-production multi-agent coding pipeline. It orchestrates **MemPalace wake-up → Phase −2 (Triage) → Phase −1 → 8** against a requirements folder, with sibling pipelines for **bug-fix** (`/architect-team:bug-fix`), **UX testing** (`/architect-team:ux-test`), and **mini** (`/architect-team:mini`).

As of **v3.39.0** it ships **48 skills + the v2.3.0 phenotype subsystem (now 4 seeded phenotypes — `user-management`, `config-management`, `ai-management`, `code-wiki`), 39 named agents, 23 slash commands, the new `services/` server tier (v3.23.0+, with the `librarian` + `triage` + `session-review` + `seeded-mempalace` services + the REPO-1…4 separation manifest — the CT6-6 program is now complete; ## Conventions at a glance

- Author commits with explicit override (repo's local git config has a "Paul Ingrram" typo):
  `git -c user.name="Paul Ingram" -c user.email="paulingram@users.noreply.github.com" commit ...`
- Run the test suite via `python -m pytest -v` from repo root (green under both Windows cp1252 and `PYTHONUTF8=1`; the expected pass/skip count for the current release is in the CHANGELOG entry).
- Bump version: update `.claude-plugin/plugin.json` + `.claude-plugin/marketplace.json` + `CHANGELOG.md`.
- Runtime state lives under `.architect-team/` (gitignored). v1.1.0 splits it into **shared** (in main worktree: `locks/`, `.mempalace/`, `run-history/`) and **per-run** (in current worktree: `reviews/`, `teammates/`, `handoffs/`, this-run's `openspec/changes/<slug>/`). The `scripts/setup/worktree_paths.py` helper resolves both via `shared_state_dir()` / `run_state_dir()`. **v1.2.0** auto-creates the run worktree (default; `--no-worktree` opts out) via `scripts/setup/worktree_lifecycle.py::create_run_worktree(slug)` — branch `architect-team/<slug>`, path `<parent-of-repo>/<repo-name>-<slug>/`. **v1.3.0** auto-cleans merged worktrees on the next run via `cleanup_merged_worktrees()` (with `exclude_current=True` keeping the cwd safe); the `/architect-team:cleanup-worktrees` command is the on-demand variant. **v3.7.0** auto-merges a clean run's branch `--no-ff` into `main` and prunes it (`--no-auto-merge` opts out; a conflict falls back to a PR, never `--force`).
