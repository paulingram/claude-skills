# architect-team plugin

## Codebase Overview

This repo IS the source of the `architect-team` Claude Code plugin — a spec-to-production multi-agent coding pipeline. As of v0.9.33 it ships **24 skills, 26 named agents, 9 slash commands**, 3 enforcement hooks, cross-platform setup scripts, and an opt-in per-project email notifier. The plugin orchestrates **MemPalace wake-up → Phase −2 (Triage) → Phase −1 → 8** against a requirements folder, with sibling pipelines for **bug-fix** (`/architect-team:bug-fix`) and **UX testing** (`/architect-team:ux-test`, v0.9.29). **v0.9.33 adds a new upstream capability — the proposal-refiner**: a conversational, codebase-grounded prompt-refinement step that runs BEFORE Phase −2 (Triage) of any pipeline when input is plain-language prose. The new **`proposal-refiner`** skill (6 phases R1-R6) + the new **`prompt-refiner`** agent (opus, read-only on source, bounded Write to `.architect-team/refined-prompts/`) load every codebase map (CODEBASE_MAP / ROUTE_MAP / DESIGN_MAP / INTERACTION_INTUITION_MAP / INTEGRATION_MAP) + a read-only MemPalace wake-up, grade the prompt on five axes (Clarity / Scope / Acceptance / Codebase grounding / Conflict — 1-10 each, weighted to 0-100 with letter A-F), and iterate with the user (5-iteration ceiling, `ship it` / `proceed` confirms termination, A-grade auto-confirms) until satisfied. The refined prompt lands as a structured markdown at `<cwd>/.architect-team/refined-prompts/<slug>-<ts>.md` with sections `## Goal` / `## Scope (in)` / `## Scope (out)` / `## Acceptance criteria` / `## Codebase touchpoints` / `## Open questions` / `## Refinement log`. Two modes: **pipeline-integrated** (downstream pipeline rebinds `$REQ_DIR` to the refined markdown) and **standalone** (via new `/architect-team:refine-prompt` command — writes markdown and exits). Classified as DOMAIN gate (v0.9.21 carve-out); `--no-refine` is the opt-out. v0.9.32 closed the v0.9.31 generalization gap — the wrong-code-path-witness discipline now lives at all 3 Playwright-running sites (B2 selector witness, Phase 5 feature code-path witness, U6 flow-effect witness). v0.9.31 added the first witness (Phase B6 `qa-replayer` Step 4.5, 4th verdict `test-did-not-exercise-fix`). v0.9.30 was a portability hot-fix — every plugin-script invocation now uses the polyglot pattern `python3 X.py args || python X.py args` so a default Windows python.org install (where only `python` is on PATH) no longer hits the Microsoft Store shim's *"Python was not found"* failure on every hook firing. v0.9.29 added two related capabilities: (1) the **`ux-test-builder` skill** + `/architect-team:ux-test` command + 2 new agents (`flow-explorer`, `flow-executor`) — a persona-driven Playwright test orchestrator that maps the target site, drafts a literal flow, expands it via 3 explorer agents proposing 10-15 adjacent flows each, executes everything in parallel via 3 executor agents against the live target, resolves disagreements via 3-cycle bounded convergence, and auto-routes failed flows through `bug-fix-pipeline`; AND (2) the **`bug-fix-pipeline` Phase B6b Logical Sensibility Check** + the new `fix-sensibility-checker` agent — closes a real-world gap (the *"auth-unavailable after Sign-Back-In fix"* case) by computing the impact set from a fix's git diff (changed files + their importers + nav destinations + endpoints) and running minimal Playwright sensibility flows against the deployed dev environment; `nonsensical` items route as fresh SRs with `origin.kind: "fix-regression"` for recursive bug-fix processing. The full chain: persona → UX flows discovered → executed → bugs found → bug-fix applied → QA-replay verifies symptom-gone → B6b sensibility-checks the impact set → any regression auto-loops → archive. **The v0.9.23 cohesion review is closed** (v0.9.24-v0.9.28).

**Stack:** Markdown (skills/agents/commands), JSON (plugin/marketplace/hooks metadata), Python 3.10+ (hooks + setup scripts + the stdlib-only `scripts/notify/` email notifier + 1139 pytest self-tests).

**Structure:** `.claude-plugin/` (identity), `skills/` (24 dirs), `agents/` (26 files), `commands/` (9 files), `hooks/` (JSON wiring + 3 enforcement scripts + a shared `review_evidence_schema.py` module — review-gate evidence schema v6), `scripts/setup/` (`setup.py` + `install_mempalace.py`), `scripts/notify/` (`notify.py` — the best-effort, stdlib-only per-project email notifier the orchestrator invokes for the five pipeline events when a project supplies `.architect-team-notify.json`; v0.9.18, wired in BOTH pipelines after v0.9.27), `tests/` (1139 pytest tests across 54 test files with structural + cross-consistency coverage), `docs/` (CODEBASE_MAP.md, INTEGRATION_MAP.md, `superpowers/` historical design + plan).

For full architecture, file purposes, conventions, gotchas, and a navigation guide, see [`docs/CODEBASE_MAP.md`](docs/CODEBASE_MAP.md).

## Conventions at a glance

- Author commits with explicit override (repo's local git config has a "Paul Ingrram" typo):
  `git -c user.name="Paul Ingram" -c user.email="paulingram@users.noreply.github.com" commit ...`
- Run the test suite via `python -m pytest -v` from repo root (1139 PASS expected as of v0.9.33).
- Bump version: update `.claude-plugin/plugin.json` + `.claude-plugin/marketplace.json` + `CHANGELOG.md`.
- Runtime state lives under `.architect-team/` (gitignored): reviews, teammates, handoffs, etc.
