# architect-team plugin

## Codebase Overview

This repo IS the source of the `architect-team` Claude Code plugin — a spec-to-production multi-agent coding pipeline. As of v0.9.34 it ships **25 skills, 26 named agents, 9 slash commands**, 3 enforcement hooks, cross-platform setup scripts, and an opt-in per-project email notifier. The plugin orchestrates **MemPalace wake-up → Phase −2 (Triage) → Phase −1 → 8** against a requirements folder, with sibling pipelines for **bug-fix** (`/architect-team:bug-fix`) and **UX testing** (`/architect-team:ux-test`, v0.9.29). **v0.9.34 adds a cross-cutting email-testing discipline** — the new **`email-testing`** skill (4 phases E1-E4) provides automatic Mailpit-based email flow verification across all QA agents. When any QA agent (`bug-replicator`, `flow-executor`, `integration`) detects email-touching work in its slice — template files, SMTP/transactional-email client imports, email-sending function calls — the discipline activates automatically: provisions Mailpit (local SMTP trap, zero external dependencies), reads the email template source to understand purpose, triggers the send via Playwright UI interaction, captures the email via Mailpit REST API, extracts every link, classifies each by purpose (invite-accept, password-reset, email-verification, unsubscribe, calendar-event, destructive-action, general-link), and follows every link in Playwright to complete the full user flow. Wired into `bug-fix-pipeline` Phase B2, `architect-team-pipeline` Phase 5, and `ux-test-builder` Phase U5/U6. No new agent or command — the skill is cross-cutting like `playwright-user-flows`. v0.9.33 added the **proposal-refiner**: a conversational, codebase-grounded prompt-refinement step that runs BEFORE Phase −2 (Triage) of any pipeline when input is plain-language prose. v0.9.32 closed the v0.9.31 generalization gap — the wrong-code-path-witness discipline now lives at all 3 Playwright-running sites (B2 selector witness, Phase 5 feature code-path witness, U6 flow-effect witness). v0.9.31 added the first witness (Phase B6 `qa-replayer` Step 4.5, 4th verdict `test-did-not-exercise-fix`). v0.9.30 was a portability hot-fix — every plugin-script invocation now uses the polyglot pattern `python3 X.py args || python X.py args` so a default Windows python.org install (where only `python` is on PATH) no longer hits the Microsoft Store shim's *"Python was not found"* failure on every hook firing. **The v0.9.23 cohesion review is closed** (v0.9.24-v0.9.28).

**Stack:** Markdown (skills/agents/commands), JSON (plugin/marketplace/hooks metadata), Python 3.10+ (hooks + setup scripts + the stdlib-only `scripts/notify/` email notifier + 1262 pytest self-tests).

**Structure:** `.claude-plugin/` (identity), `skills/` (25 dirs), `agents/` (26 files), `commands/` (9 files), `hooks/` (JSON wiring + 3 enforcement scripts + a shared `review_evidence_schema.py` module — review-gate evidence schema v6), `scripts/setup/` (`setup.py` + `install_mempalace.py`), `scripts/notify/` (`notify.py` — the best-effort, stdlib-only per-project email notifier the orchestrator invokes for the five pipeline events when a project supplies `.architect-team-notify.json`; v0.9.18, wired in BOTH pipelines after v0.9.27), `tests/` (1262 pytest tests across 57 test files with structural + cross-consistency coverage), `docs/` (CODEBASE_MAP.md, INTEGRATION_MAP.md, `superpowers/` historical design + plan).

For full architecture, file purposes, conventions, gotchas, and a navigation guide, see [`docs/CODEBASE_MAP.md`](docs/CODEBASE_MAP.md).

## Conventions at a glance

- Author commits with explicit override (repo's local git config has a "Paul Ingrram" typo):
  `git -c user.name="Paul Ingram" -c user.email="paulingram@users.noreply.github.com" commit ...`
- Run the test suite via `python -m pytest -v` from repo root (1262 PASS expected as of v0.9.34).
- Bump version: update `.claude-plugin/plugin.json` + `.claude-plugin/marketplace.json` + `CHANGELOG.md`.
- Runtime state lives under `.architect-team/` (gitignored): reviews, teammates, handoffs, etc.
