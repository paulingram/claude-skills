# architect-team plugin

## Codebase Overview

This repo IS the source of the `architect-team` Claude Code plugin ŌĆö a spec-to-production multi-agent coding pipeline. As of v0.9.32 it ships **23 skills, 25 named agents, 8 slash commands**, 3 enforcement hooks, cross-platform setup scripts, and an opt-in per-project email notifier. The plugin orchestrates **MemPalace wake-up ŌåÆ Phase ŌłÆ2 (Triage) ŌåÆ Phase ŌłÆ1 ŌåÆ 8** against a requirements folder, with sibling pipelines for **bug-fix** (`/architect-team:bug-fix`) and **UX testing** (`/architect-team:ux-test`, v0.9.29). **v0.9.32 closes the v0.9.31 generalization gap** ŌĆö the wrong-code-path-witness discipline now lives at ALL THREE Playwright-running sites instead of one: (1) `bug-replicator` Phase B2 gets a **selector witness** (`.toBeVisible()` + `.toBeEnabled()` + disambiguating role/attribute check before every action) catching the failure at AUTHORING time; (2) `integration` agent Phase 5 gets the **code-path execution witness** adapted to features (reads `implementing_commits[]`) with verdict `feature-tests-did-not-exercise-implementation`; (3) `flow-executor` Phase U6 gets a **flow-effect witness** verifying the U5-authored `expected_user_effect` (DOM/network/URL/console) ŌĆö failing flows route as `origin.kind: "flow-effect-gap"`. v0.9.31 added the first witness instance (Phase B6's `qa-replayer` Step 4.5) with the 4th verdict `test-did-not-exercise-fix`. v0.9.30 was a portability hot-fix ŌĆö every plugin-script invocation now uses the polyglot pattern `python3 X.py args || python X.py args` so a default Windows python.org install (where only `python` is on PATH) no longer hits the Microsoft Store shim's *"Python was not found"* failure on every hook firing. v0.9.29 added two related capabilities: (1) the **`ux-test-builder` skill** + `/architect-team:ux-test` command + 2 new agents (`flow-explorer`, `flow-executor`) ŌĆö a persona-driven Playwright test orchestrator that maps the target site, drafts a literal flow, expands it via 3 explorer agents proposing 10-15 adjacent flows each, executes everything in parallel via 3 executor agents against the live target, resolves disagreements via 3-cycle bounded convergence, and auto-routes failed flows through `bug-fix-pipeline`; AND (2) the **`bug-fix-pipeline` Phase B6b Logical Sensibility Check** + the new `fix-sensibility-checker` agent ŌĆö closes a real-world gap (the *"auth-unavailable after Sign-Back-In fix"* case) by computing the impact set from a fix's git diff (changed files + their importers + nav destinations + endpoints) and running minimal Playwright sensibility flows against the deployed dev environment; `nonsensical` items route as fresh SRs with `origin.kind: "fix-regression"` for recursive bug-fix processing. The full chain: persona ŌåÆ UX flows discovered ŌåÆ executed ŌåÆ bugs found ŌåÆ bug-fix applied ŌåÆ QA-replay verifies symptom-gone ŌåÆ B6b sensibility-checks the impact set ŌåÆ any regression auto-loops ŌåÆ archive. **The v0.9.23 cohesion review is closed** (v0.9.24-v0.9.28).

**Stack:** Markdown (skills/agents/commands), JSON (plugin/marketplace/hooks metadata), Python 3.10+ (hooks + setup scripts + the stdlib-only `scripts/notify/` email notifier + 1042 pytest self-tests).

**Structure:** `.claude-plugin/` (identity), `skills/` (23 dirs), `agents/` (25 files), `commands/` (8 files), `hooks/` (JSON wiring + 3 enforcement scripts + a shared `review_evidence_schema.py` module ŌĆö review-gate evidence schema v6), `scripts/setup/` (`setup.py` + `install_mempalace.py`), `scripts/notify/` (`notify.py` ŌĆö the best-effort, stdlib-only per-project email notifier the orchestrator invokes for the five pipeline events when a project supplies `.architect-team-notify.json`; v0.9.18, wired in BOTH pipelines after v0.9.27), `tests/` (1042 pytest tests across 50 test files with structural + cross-consistency coverage), `docs/` (CODEBASE_MAP.md, INTEGRATION_MAP.md, `superpowers/` historical design + plan).

For full architecture, file purposes, conventions, gotchas, and a navigation guide, see [`docs/CODEBASE_MAP.md`](docs/CODEBASE_MAP.md).

## Conventions at a glance

- Author commits with explicit override (repo's local git config has a "Paul Ingrram" typo):
  `git -c user.name="Paul Ingram" -c user.email="paulingram@users.noreply.github.com" commit ...`
- Run the test suite via `python -m pytest -v` from repo root (1042 PASS expected as of v0.9.32).
- Bump version: update `.claude-plugin/plugin.json` + `.claude-plugin/marketplace.json` + `CHANGELOG.md`.
- Runtime state lives under `.architect-team/` (gitignored): reviews, teammates, handoffs, etc.
