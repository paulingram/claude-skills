# architect-team plugin

## Codebase Overview

This repo IS the source of the `architect-team` Claude Code plugin — a spec-to-production multi-agent coding pipeline. As of v1.0.0 it ships **26 skills, 27 named agents, 11 slash commands**, 3 enforcement hooks (now wired for both `PostToolUse(TaskUpdate)` / `SubagentStop` and the teams-mode `TaskCompleted` / `TeammateIdle` triggers), cross-platform setup scripts, an opt-in per-project email notifier, a new cross-session lock layer (`hooks/locks.py`), and a new teams-mode detection helper (`scripts/setup/teams_mode.py`). **v1.0.0 is the structural pivot to Claude Code's experimental Agent Teams primitive as the default dispatch mode** — long-lived named teammates with their own 1M context windows, a shared task list, direct `SendMessage`, and a Lead that owns coordination. The Lead is the listening point the user has been asking for; the shared task list IS the parallel-marshalling primitive. Teams mode requires the `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` env var (or the equivalent in `~/.claude/settings.json`) AND Claude Code ≥ 2.1.32 — without those, the pipeline silently falls back to subagents mode (the v0.10.0 behavior, unchanged). A `--no-teams` flag on any of the three pipeline commands forces subagents mode even when teams are available. Every nested-team pattern in the pipeline (`task-reviewer ×3`, `editability-reviewer ×3`, `interaction-reviewer ×3`, `integration-explorer ×3 + master-synthesizer`, `visual-capture + visual-analyzer`, `diagnostic-researcher ×3`, `codebase-map-reviewer ×3`, `flow-explorer ×3`, `flow-executor ×3`) flattens to Lead-owned dispatch per the Agent Teams docs' "no nested teams" constraint. The plugin orchestrates **MemPalace wake-up → Phase −2 (Triage) → Phase −1 → 8** against a requirements folder, with sibling pipelines for **bug-fix** (`/architect-team:bug-fix`), **UX testing** (`/architect-team:ux-test`, v0.9.29), and **mini** (`/architect-team:mini`, v0.10.0). (v0.10.0 introduced the mini-architect-team-pipeline — single architect, single `mini-qa` agent, parallel devs cross-reviewing each other, auto-merge to `main` on green QA, `Mini-Run:` commit trailer + the `/architect-team:mini-review-sweep` batch replay.)

**Stack:** Markdown (skills/agents/commands), JSON (plugin/marketplace/hooks metadata), Python 3.10+ (hooks + setup scripts + the new `scripts/setup/teams_mode.py` detector + the new `hooks/locks.py` cross-session lock layer + the stdlib-only `scripts/notify/` email notifier + ~1629 pytest self-tests).

**Structure:** `.claude-plugin/` (identity), `skills/` (26 dirs), `agents/` (27 files), `commands/` (11 files), `hooks/` (JSON wiring + 3 enforcement scripts + a shared `review_evidence_schema.py` module — review-gate evidence schema v6 — + the new `locks.py` cross-session lock layer), `scripts/setup/` (`setup.py` + `install_mempalace.py` + the new `teams_mode.py` detector), `scripts/notify/` (`notify.py` — the best-effort, stdlib-only per-project email notifier the orchestrator invokes for the five pipeline events when a project supplies `.architect-team-notify.json`; v0.9.18, wired in BOTH pipelines after v0.9.27), `tests/` (~1629 pytest tests across 72 test files with structural + cross-consistency coverage), `docs/` (CODEBASE_MAP.md, INTEGRATION_MAP.md, `superpowers/` historical design + plan).

For full architecture, file purposes, conventions, gotchas, and a navigation guide, see [`docs/CODEBASE_MAP.md`](docs/CODEBASE_MAP.md).

## Conventions at a glance

- Author commits with explicit override (repo's local git config has a "Paul Ingrram" typo):
  `git -c user.name="Paul Ingram" -c user.email="paulingram@users.noreply.github.com" commit ...`
- Run the test suite via `python -m pytest -v` from repo root (~1629 PASS expected as of v1.0.0).
- Bump version: update `.claude-plugin/plugin.json` + `.claude-plugin/marketplace.json` + `CHANGELOG.md`.
- Runtime state lives under `.architect-team/` (gitignored): reviews, teammates, handoffs, etc.
