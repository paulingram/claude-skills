# architect-team plugin

## Codebase Overview

This repo IS the source of the `architect-team` Claude Code plugin — a spec-to-production multi-agent coding pipeline. As of v0.9.30 it ships **23 skills, 25 named agents, 8 slash commands**, 3 enforcement hooks, cross-platform setup scripts, and an opt-in per-project email notifier. The plugin orchestrates **MemPalace wake-up → Phase −2 (Triage) → Phase −1 → 8** against a requirements folder, with sibling pipelines for **bug-fix** (`/architect-team:bug-fix`) and **UX testing** (`/architect-team:ux-test`, v0.9.29). **v0.9.30 is a portability hot-fix** — every plugin-script invocation in `hooks/hooks.json`, in skill bodies (`architect-team-pipeline`, `bug-fix-pipeline`), and in the 4 commands (`architect-team`, `bug-fix`, `ux-test`, `architect-team-setup`, `mempalace-install`) now uses the polyglot pattern `python3 X.py args || python X.py args` so a default Windows python.org install (where only `python` is on PATH) no longer hits the Microsoft Store shim's *"Python was not found"* failure on every hook firing. v0.9.29 added two related capabilities: (1) the **`ux-test-builder` skill** + `/architect-team:ux-test` command + 2 new agents (`flow-explorer`, `flow-executor`) — a persona-driven Playwright test orchestrator that maps the target site, drafts a literal flow, expands it via 3 explorer agents proposing 10-15 adjacent flows each, executes everything in parallel via 3 executor agents against the live target, resolves disagreements via 3-cycle bounded convergence, and auto-routes failed flows through `bug-fix-pipeline`; AND (2) the **`bug-fix-pipeline` Phase B6b Logical Sensibility Check** + the new `fix-sensibility-checker` agent — closes a real-world gap (the *"auth-unavailable after Sign-Back-In fix"* case) by computing the impact set from a fix's git diff (changed files + their importers + nav destinations + endpoints) and running minimal Playwright sensibility flows against the deployed dev environment; `nonsensical` items route as fresh SRs with `origin.kind: "fix-regression"` for recursive bug-fix processing. The full chain: persona → UX flows discovered → executed → bugs found → bug-fix applied → QA-replay verifies symptom-gone → B6b sensibility-checks the impact set → any regression auto-loops → archive. **The v0.9.23 cohesion review is closed** (v0.9.24-v0.9.28).

**Stack:** Markdown (skills/agents/commands), JSON (plugin/marketplace/hooks metadata), Python 3.10+ (hooks + setup scripts + the stdlib-only `scripts/notify/` email notifier + 1016 pytest self-tests).

**Structure:** `.claude-plugin/` (identity), `skills/` (23 dirs), `agents/` (25 files), `commands/` (8 files), `hooks/` (JSON wiring + 3 enforcement scripts + a shared `review_evidence_schema.py` module — review-gate evidence schema v6), `scripts/setup/` (`setup.py` + `install_mempalace.py`), `scripts/notify/` (`notify.py` — the best-effort, stdlib-only per-project email notifier the orchestrator invokes for the five pipeline events when a project supplies `.architect-team-notify.json`; v0.9.18, wired in BOTH pipelines after v0.9.27), `tests/` (1016 pytest tests across 50 test files with structural + cross-consistency coverage), `docs/` (CODEBASE_MAP.md, INTEGRATION_MAP.md, `superpowers/` historical design + plan).

For full architecture, file purposes, conventions, gotchas, and a navigation guide, see [`docs/CODEBASE_MAP.md`](docs/CODEBASE_MAP.md).

## Conventions at a glance

- Author commits with explicit override (repo's local git config has a "Paul Ingrram" typo):
  `git -c user.name="Paul Ingram" -c user.email="paulingram@users.noreply.github.com" commit ...`
- Run the test suite via `python -m pytest -v` from repo root (1016 PASS expected as of v0.9.30).
- Bump version: update `.claude-plugin/plugin.json` + `.claude-plugin/marketplace.json` + `CHANGELOG.md`.
- Runtime state lives under `.architect-team/` (gitignored): reviews, teammates, handoffs, etc.
