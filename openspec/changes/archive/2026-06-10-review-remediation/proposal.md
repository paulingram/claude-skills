## Why

A 2026-06-09 codebase review of CLAUDE TEAM SIX (v3.9.2) found **30 verified defects** across the enforcement layer, the command surface, the skill bodies, and the documentation. Each was independently confirmed against the working tree at commit `f2510a7` with `file:line` evidence (see `.architect-team/requirements/review-remediation/REQUIREMENTS.md`). The defects fall into a single class: **the plugin's machinery silently fails or silently mis-teaches**, so the failure never surfaces until a downstream run is already broken.

Representative confirmed defects:

- **The hook wiring double-executes or silently drops blocks.** Every command in `hooks/hooks.json` is `python3 X || python X`; when a hook BLOCKS with exit 2, the `||` re-runs it (double execution + double BLOCKED message) or, on a `python3`-only host, exits 127 so the block is silently dropped. The v2.16.0 detect-once fix already shipped in `commands/architect-team.md:175` but was never propagated to the hook wiring itself.
- **Three VAO CLI tools crash as scripts.** `verify-discipline-registry-current`, `verify-inflight-clarifications-processed`, and `verify-no-unilateral-override` raise `ModuleNotFoundError` when invoked as `python hooks/vao_tools.py <subcommand>` because of package-only lazy imports with no bare-module fallback — even though the file's own top-of-module imports (lines 61–68) already model the dual-form pattern.
- **Windows-path false negatives degrade gates to no-ops.** `verify_no_pipeline_bypass` matches `/reviews/` against a raw backslash ledger path → false `independent-review-bypassed`; four hook scripts decode stdin with the locale codec → a UTF-8 task title degrades the gate to a silent no-op under cp1252; two hooks catch `JSONDecodeError` but not `OSError` → a Windows sharing-violation tracebacks the gate into a silent skip.
- **The `/architect-team:inject` headline feature can lose messages.** `inflight_inbox.py::mark_processed` does a non-atomic read-all/rewrite-all that can destroy a concurrently-appended message from a separate-terminal inject, and `run_id` reaches the inbox filename without the `safe_id` path-traversal validation its sibling hooks apply.
- **Five command banners and two cleanup calls are silent no-ops.** `teams_mode.py` and `worktree_lifecycle.py` are invoked with CLI flags they don't implement (no `__main__`), so the v1.5.0 dispatch banner and the v1.3.0 worktree cleanup never run from those commands.
- **The docs mis-teach the system.** The evidence schema is taught as **v6** in six places while `hooks/review_evidence_schema.py` is ground-truth **v7** with **17 required fields**; v3.8.0 unbounded-solving residue still claims "bounded at 3 cycles" in eight skill/command locations; a canonical MemPalace not-on-PATH note is referenced from four pipeline bodies but **does not exist**; seven skill descriptions exceed the 1024-char Agent Skills limit; and the project docs contradict themselves on test counts (3871 vs 3928), VAO-tool counts (7 vs 14 vs the real 20), and enforcement-script counts (3 vs the real 4).

The owner directive is binding and full-build: **implement every item A1–A10, B1–B5, C1–C6, D1–D4, E1–E3 in full — no scope cuts, no deferrals, no "foundation only".** The documentation reconciliation (D) is in-scope work for THIS run, not a follow-up. A regression-test family (E1) that executes the real glue would have caught A1/A2/A5/A6/B1/C2 before they shipped; it is part of this change so none of these re-ships.

No gate is relaxed; the hook gates become *more* reliable (they stop double-firing and stop silently no-opping), and the schema documentation stops under-describing the v7 contract the code already enforces.

### Playwright Flows

**Zero Playwright flows.** This codebase is a Claude Code plugin with no web UI, no rendered pages, and no dev URL — there is nothing for a browser to drive. The required `### Playwright Flows` subsection is present (structural-contract requirement) but intentionally enumerates no flows; the browser-flow verification it normally represents is replaced, for this plugin, by the E1 CLI/hook subprocess-execution family above. This adaptation is documented in `design.md` under "QA-contract adaptation for a no-UI codebase". mini-qa verifies behavior via pytest + real CLI execution, not a browser.

### Out of Scope

QA MUST NOT:
- Do any live web testing, browser automation, or dev-server interaction — there is no web surface.
- Touch any production system, deployed URL, or external service — none exist for this plugin.
- Run destructive git operations (`stash` / `reset --hard` / `rebase` / `commit --amend` / `checkout` of another branch / `clean -f`) — the run is shared-tree-disciplined.
- Modify any file outside the enumerated A–E scope, or broaden a gate's behavior beyond restoring its documented contract (A10 specifically forbids broadening the matcher while fixing it).
- Add or change runtime third-party dependencies — Python stays stdlib-only.
