## 1. System Overview

The `architect-team` Claude Code plugin (v0.9.0) is a spec-to-production multi-agent coding pipeline. It accepts a requirements folder (in OpenSpec, Superpowers, or plain-markdown format) and drives it end-to-end through intake mapping (with conditional design-fidelity mapping when screenshots / Figma / tokens / Storybook / assets are present), 100%-coverage planning validation, parallel team implementation with mandatory review gates (now including hook-enforced visual-fidelity reconciliation against DESIGN_MAP.md when frontend was touched ‚Äî drift is fixed-to-spec autonomously by default; AND hook-enforced test-completeness verification confirming unit + integration + Playwright tests all ran and pass a grep-audit for forbidden API-direct-call patterns), reconciliation, cross-layer integration testing with mandatory per-test expectations and a 3-pass root-cause loop on every failure plus visual-fidelity regression sweep across all designed screens, and a final coverage-verified report. The plugin ships 11 skills, 11 named agent definitions, 3 slash commands, 2 enforcement hooks, a cross-platform setup script, and 101 pytest self-tests that validate every structural artifact.

## 4. Module Guide

### Skills

**`skills/architect-team-pipeline/SKILL.md`** ‚Äî Orchestrator (Phase ‚àí1 ‚Üí 8 playbook).
Frontmatter: `name: architect-team-pipeline`, `argument-hint: [path-to-requirements-folder]`. No `disable-model-invocation` (removed in v0.2.1).
Dependencies: every auxiliary skill + every agent + `openspec` CLI + `cartographer`/`ralph-loop`/`superpowers` plugins. Dependents: `commands/architect-team.md`. Gotcha: must NOT have `disable-model-invocation: true`; must NOT be named `architect-team` (name collision).

**`skills/intake-and-mapping/SKILL.md`** ‚Äî Phase ‚àí1 workflow. Codebase discovery (4-priority), per-codebase ralph loop (exit `"CODEBASE MAP COMPLETE"`, `--max-iterations 10`), integration ralph loop (exit `"INTEGRATION MAP COMPLETE"`, `--max-iterations 8`). Persists state to `.architect-team/intake-state.json`. Gotcha: integration mapping runs even for single-codebase work.

**`skills/reuse-first-design/SKILL.md`** ‚Äî Extend > compose > reuse > build-new ladder. Every `design.md` must have `## Reuse Decisions` section with one entry per new module/file/dependency. Phase 1 cross-checks Reuse Decisions against actual CODEBASE_MAP.md symbols.

**`skills/frontend-route-mapping/SKILL.md`** ‚Äî ROUTE_MAP.md schema + completeness rubric. Frontmatter on ROUTE_MAP: `last_routed`, `codebase`, `framework`. Five required body sections including the navigation web. Freshness via `git log -1 --format=%cI`.

**`skills/playwright-user-flows/SKILL.md`** ‚Äî Two-phase methodology: Examine (read code, build interactivity inventory JSON) ‚Üí Author (one test per inventory entry, selector hierarchy role>testid>text>css) ‚Üí Verify (playwright-coverage.json with all inventory IDs accounted for). API-only testing is explicitly rejected.

**`skills/dev-api-integration-testing/SKILL.md`** ‚Äî Live-dep integration tests; three assertion layers (response shape, side-effect verification, audit/log); per-test data prefix `it-<test_name>-<uuid>`; cover every documented error response; idempotent.

**`skills/coverage-mapping/SKILL.md`** ‚Äî `coverage-map.json` schema. Used Phase 1 (validation), Phase 3 (per-team slice), Phase 7 (master review), Phase 8 (final report spine). Atomic write pattern (.tmp + rename).

**`skills/team-spawning-and-review-gates/SKILL.md`** ‚Äî Non-overlapping file scopes, plan-approval-mode triggers (auth/schema/contracts/external/secrets), handoff files as primary coordination (`.architect-team/handoffs/<from>-to-<to>-<timestamp>.md`), review-gate evidence schema (8 required fields), teammate manifest schema (6 fields).

**`skills/root-cause-test-failures/SKILL.md`** ‚Äî v0.3.0. Three disciplines: predict before testing (`<test-output-dir>/expectations/<test-id>.json`), refuse to rationalize (every cause must be evidence-backed), three passes minimum (forward data-flow trace ‚Üí backward call-flow trace ‚Üí alternative-hypotheses sweep). Produces structured `<test-output-dir>/rca/<test-id>-<ts>.json` with file:line evidence at every hypothesis. Phase C escalation: `product-bug` ‚Üí handoff to architect; `test-author-error` ‚Üí correct expectation; `env/fixture/race/cache` ‚Üí document trigger + fix + prevention. Validated by RED/GREEN pressure tests; GREEN caught a second defect RED missed entirely.

**`skills/design-fidelity-mapping/SKILL.md`** ‚Äî v0.4.0. Conditional skill that activates when design artifacts exist (screenshots / Figma / `tailwind.config` / `theme.ts` / `tokens.json` / Storybook / brand docs / `assets/` directory). Produces `<codebase>/docs/DESIGN_MAP.md` with: Design Tokens tables (color palette, typography scale, spacing, radii, shadows, breakpoints ‚Äî all with source citations), Asset Registry (every static image with path / dimensions / SHA-256 hash / variants / where-referenced), Per-Screen Visual Specs (per-element computed-style spec table for every interactive element on every designed screen, plus asset-placement diagrams and responsive breakpoint deltas), Theme Variants, Detected Drift (every disagreement between design and implementation captured explicitly), Coverage & Gaps (with `escalate: true` flag for the orchestrator). Skipped (correctly) when no design inputs exist.

**`skills/visual-fidelity-reconciliation/SKILL.md`** ‚Äî NEW v0.5.0. Hook-enforced strict post-development QA discipline against DESIGN_MAP.md. Zero-tolerance defaults (0px / exact color / exact font / exact spacing). Exhaustive state walks (default / hover / focus / active / disabled / loading / error / empty + every responsive viewport). Phase B code-first static analysis (resolve every styling layer to its concrete value and compare to spec; verify asset SHA-256s). Phase C runtime verification (Playwright at each viewport, induce every state, capture per-state element screenshot + per-viewport full-page screenshot). Phase D reconciliation JSON per (screen, viewport). **Phase E remediation: fix-to-spec by default; escalate ONLY on four named cases ‚Äî out-of-scope file, implementation-has-element-not-in-spec, spec-ambiguity, cascade-blast-radius.** Enforced via the `visual_fidelity_review` evidence field in the review-gate hook (`"pass"` / `"n/a"` / `"fail"`; `"fail"` blocks completion).

### Commands

**`commands/architect-team.md`** ‚Äî User entry `/architect-team:architect-team <path>`. Thin wrapper that delegates to `architect-team-pipeline` skill via the Skill tool. **Gotcha**: command-level `$ARGUMENTS` does NOT propagate into the invoked skill body (the skill sees empty `$ARGUMENTS`).

**`commands/architect-team-setup.md`** ‚Äî User entry `/architect-team:architect-team-setup [--check-only] [--force-reinstall]`. Runs `${CLAUDE_PLUGIN_ROOT}/scripts/setup/setup.py` via a `!` shell block.

**`commands/visual-qa.md`** ‚Äî NEW v0.5.0. User entry `/architect-team:visual-qa [<codebase-path>]`. On-demand visual-fidelity audit. Discovers frontend codebases from intake-state.json or `$ARGUMENTS`, freshness-checks DESIGN_MAP.md (refreshes via route-mapper if stale), runs visual-fidelity-reconciliation across all designed screens with fix-to-spec by default. Emits structured PASS / DRIFT_DETECTED / GAPS_DETECTED report with handoff paths.

### Hooks

**`hooks/hooks.json`** ‚Äî Wires `PostToolUse[matcher=TaskUpdate]` ‚Üí `review-gate-task.py` and `SubagentStop[matcher=*]` ‚Üí `teammate-idle-check.py`. Both `async: false`. Both use `${CLAUDE_PLUGIN_ROOT}`.

**`hooks/review-gate-task.py`** ‚Äî Entry: `main()`. Reads stdin JSON; checks `tool_name == "TaskUpdate"` and `status == "completed"`; validates `<cwd>/.architect-team/reviews/<taskId>.json`. Exit 0 = allow, exit 2 = block. Validates 10 required fields (v0.9.0 added `test_completeness_review`; v0.5.0 added `visual_fidelity_review`) with specific rules (spec_review/quality_review/reuse_compliance = pass values, tests.added ‚â• 1, visual_fidelity_review ‚àà {pass, n/a, fail} with `"fail"` blocked and `"n/a"` requiring a non-empty note, test_completeness_review ‚àà {pass, n/a, fail} with same pattern). Malformed hook payload ‚Üí exit 0 (infrastructure error tolerance).

**`hooks/teammate-idle-check.py`** ‚Äî Entry: `main()`. Extracts subagent name (handles both nested `subagent.name` and flat `subagent_name` payload shapes); reads `.architect-team/teammates/<name>.json`; validates each `expected_review_evidence` task ID. No manifest = not an architect-team teammate = exit 0. Shares `_validate` schema with review-gate-task.py (duplicated, not imported).

### Setup script

**`scripts/setup/setup.py`** ‚Äî Entry: `main(argv)`. Checks Python ‚â• 3.10 and Node ‚â• 20.19; installs openspec CLI (npm), pytest+pytest-asyncio+httpx, Playwright+chromium; checks for required Claude plugins. Exit codes: 0=all present, 1=plugin missing, 2=install failure. **`_install_packages` adds `--system` to `uv pip install` when no venv detected** (the v0.1.1 fix). `_playwright_browser_installed` checks both the Python package AND the chromium browser cache directory.

```mermaid
    Orch->>Teammate: brief (task_ids + scope + acceptance)
    Teammate->>Teammate: implement + test
    Teammate->>FS: .architect-team/reviews/<task-id>.json
    Teammate->>HookPTU: TaskUpdate(status=completed)
    HookPTU->>FS: read evidence
    alt valid
        HookPTU-->>Teammate: exit 0
    else invalid
        HookPTU-->>Teammate: exit 2, structured stderr
        Teammate->>Teammate: fix, retry
    end
    Teammate->>HookSS: SubagentStop
    HookSS->>FS: read manifest + all evidence
    alt all valid
        HookSS-->>Teammate: exit 0
    else gaps
        HookSS-->>Teammate: exit 2, re-engage
    end
    Teammate-->>Skill: Phase 3 gate passed
    Skill->>Skill: Phases 4-8
```

## 6. Conventions

**Skill frontmatter:** required `name` (matches dir), `description` (‚â• 20 chars). Optional `argument-hint`, `disable-model-invocation` (currently absent from all skills post-v0.2.1).

**Agent frontmatter:** all 5 keys are required-by-test-presence: `name`, `description`, `tools` (comma-separated from a 13-tool valid set, validated against the set), `model` (validated against `{opus, sonnet, haiku}`), `color`. **Test-enforced validation:** `name` matches dir, `description` ‚â• 20 chars, `tools` ‚äÜ valid set + non-empty, `model` ‚àà valid set, body non-empty. **NOT test-enforced:** `color` value (presence only ‚Äî the suggested palette `blue/cyan/green/orange/magenta/purple/red` is convention, not validation). Model pattern: opus for synthesis/judgment, sonnet for implementers/reviewers, haiku for narrow mechanical (none shipped yet).

**Command frontmatter:** required `description`. Optional `argument-hint`, `allowed-tools` (Bash filter strings).

**Map artifacts:**
- `<codebase>/docs/CODEBASE_MAP.md` ‚Äî cartographer's output (`last_mapped` ISO 8601 frontmatter).
- `<codebase>/docs/ROUTE_MAP.md` ‚Äî route-mapper's output (`last_routed`, `codebase`, `framework` frontmatter).
- `<workspace>/docs/INTEGRATION_MAP.md` ‚Äî master-synthesizer's output (`last_synthesized`, `codebases`, `source_drafts` frontmatter).

**Runtime state (gitignored under `.architect-team/`):**
- `intake-state.json` ‚Äî re-entry short-circuit.
- `reviews/<task-id>.json` ‚Äî review-gate evidence.
- `teammates/<name>.json` ‚Äî teammate manifests.
- `handoffs/<from>-to-<to>-<timestamp>.md` ‚Äî primary coordination primitive.
- `reconciliation-reports/<timestamp>.md` ‚Äî reconciler output.
- `integration-drafts/explorer-<N>.md` ‚Äî Phase ‚àí1C convergence drafts.

**Test conventions:** all in `tests/`, discovered via `test_*.py` glob. Session fixtures in `conftest.py`. Parametrize pattern with `sorted(EXPECTED_*)` for stable ordering. `pytest.skip` for parametrized tests where set may outpace files. Subprocess tests use `tmp_path` workspace + `PYTHONIOENCODING=utf-8`. Setup script imported via `importlib.util` (avoids package-layout dep). `--strict-markers` enforced.

**Review-gate evidence schema.**

The 10 fields enforced by `REQUIRED_EVIDENCE_FIELDS` in `hooks/review-gate-task.py` (verified by `_validate()`):

| Field | Required value |
|---|---|
| `task_id` | non-empty string |
| `spec_review` | `"pass"` |
| `quality_review` | `"pass"` |
| `real_not_stubbed` | `true` (boolean) |
| `tests` | object containing `added` (int ‚â• 1) and `passing` (int, must equal `added`) |
| `demo_artifact` | non-empty string |
| `files_changed` | non-empty array |
| `reuse_compliance` | `"ok"` |
| `visual_fidelity_review` | `"pass"` / `"n/a"` (non-empty `_note` required for `"n/a"`) ‚Äî `"fail"` blocks |
| `test_completeness_review` | `"pass"` / `"n/a"` (non-empty `_note` required for `"n/a"`) ‚Äî `"fail"` blocks |

The schema example in `team-spawning-and-review-gates/SKILL.md` ALSO carries three conventional fields the hook does NOT enforce: `schema_version` (int, currently v3), `teammate` (string), `completed_at` (ISO 8601 string). Teammates are expected to write all 13 fields for human-readable evidence; the hook only blocks on missing/invalid required fields, not the conventional ones.

## 7. Gotchas (cross-cutting)

**v0.1.1 ‚Äî `uv pip install` requires `--system` outside a venv.** Setup script preferred `uv` over plain pip; without an active venv, `uv pip install` refused to install (breaking Playwright). Fix: detect venv via `VIRTUAL_ENV` + `sys.real_prefix` + `sys.base_prefix != sys.prefix`; add `--system` when none active.

**v0.2.0 ‚Äî skill/command name collision silently broke the pipeline.** Skill name `architect-team` collided with command name `architect-team`; Skill tool resolved to the command body (thin wrapper) instead of the Phase ‚àí1 ‚Üí 8 playbook. The pipeline appeared to start but immediately exited. Fix: renamed skill to `architect-team-pipeline`.

**v0.2.1 ‚Äî `disable-model-invocation: true` blocks the Skill tool.** The flag prevented the Skill tool from loading the orchestrator body; the command's delegation chain (Skill tool ‚Üí orchestrator) failed with "cannot be used due to disable-model-invocation". Fix: removed the flag entirely.

**v0.2.2 ‚Äî `review-gate-task.py` blocked every TaskUpdate, not just teammate ones (REQ-007).** Discovered while dogfooding the pipeline against this very repo: the orchestrator's internal task tracking via TaskUpdate triggered the review gate, which had no way to tell teammate tasks from non-teammate ones. Fix: new `_is_teammate_task(task_id, cwd)` helper walks `.architect-team/teammates/*.json` and returns true only if the task is in some manifest's `expected_review_evidence`. Absent teammates dir = no architect-team workflow = hard allow. Tests `test_exits_zero_when_task_not_in_any_manifest` and `test_exits_zero_when_no_teammates_dir` cover the new behavior; existing tests now write a manifest before exercising the gate.

**Hook exit-2 vs exit-1 semantics.** Both hooks return exit 2 (not 1) to block. Exit 0 = allow. Exit 1 is treated as error, not block. Never return exit 1 from these hooks for intentional blocking.

**Malformed hook payloads ‚Üí exit 0.** Both hooks tolerate malformed JSON on stdin (treats as infrastructure error, not teammate failure). Only the EVIDENCE file being malformed triggers exit 2.

**Teammate manifest absence = silent allow.** If `SubagentStop` fires for a subagent without a corresponding `.architect-team/teammates/<name>.json` manifest, the hook returns exit 0. Not a blocking error; just treated as non-architect-team.

**`ROUTE_MAP.md` staleness is a test blocker.** Both `playwright-user-flows` skill and `integration` agent explicitly require re-mapping before authoring tests against a stale ROUTE_MAP. Tests built on stale assumptions are treated as worse than no tests.

**Atomic writes for `coverage-map.json`.** Updated across many phases; the skill specifies write-to-`.tmp` then rename to avoid corruption on crash.

**Plan-approval mode is not retroactive.** Auth/schema/contract teammates must be spawned in plan-approval mode BEFORE any tool calls. There is no retroactive approval mechanism.

**$ARGUMENTS does not propagate from command to invoked skill** (UX wart ‚Äî **closed in v0.2.3 REQ-001**). `commands/architect-team.md` now explicitly instructs the model to bind `$REQ_DIR` from the command's `$ARGUMENTS` rather than re-prompt the user. Historical context retained because the same issue applies to any new command that wraps a skill.

**Hook scripts construct paths from unsanitized identifiers** (defensive gap ‚Äî **closed in v0.2.3 REQ-002**). Both `hooks/review-gate-task.py` and `hooks/teammate-idle-check.py` now call `_safe_id()` to reject empty strings, `/`, `\`, leading `.`, and exact `..` before constructing filesystem paths. Rejection emits structured stderr and exit 2.

**Scaffold-agent does not update EXPECTED_AGENTS.** Generated agents have valid frontmatter but the presence test (`test_all_expected_agents_present`) won't catch their absence until the set is manually updated.

## 8. Navigation Guide

**Add a new skill:** create `skills/<name>/SKILL.md` with frontmatter ‚Üí add to `EXPECTED_SKILLS` in `tests/test_skills.py` ‚Üí run `pytest tests/test_skills.py -v` ‚Üí reference from `architect-team-pipeline/SKILL.md` if pipeline-participating.

**Add a new agent:** create `agents/<name>.md` with all 5 frontmatter keys (tools from valid 13-tool set, model from {opus,sonnet,haiku}, color from valid set; quote description in YAML if it contains colons) and required body sections (intro, Boundaries, Reuse-First Mandate, Process, Hard rules) ‚Üí add to `EXPECTED_AGENTS` in `tests/test_agents.py` ‚Üí run `pytest tests/test_agents.py -v`. Or invoke `scaffold-agent` and add to set afterward.

**Add a new slash command:** create `commands/<name>.md` with `description` frontmatter ‚Üí add to `EXPECTED_COMMANDS` in `tests/test_commands.py`. Add `allowed-tools` if needed.

**Add a new hook:** write `hooks/<name>.py` (read stdin JSON; write stderr; exit 0 or 2) ‚Üí register in `hooks/hooks.json` under appropriate event with matcher and `${CLAUDE_PLUGIN_ROOT}` path ‚Üí add `tests/test_<name>.py` subprocess tests covering both exit paths + update `tests/test_hooks_structure.py` for the new wire.

**Update setup script:** edit `scripts/setup/setup.py` (`ensure_*` function pattern). Update `tests/test_setup_script.py` via `importlib` mocks. New deps follow `(name, status, detail)` tuple return.

**Bump version & release:** update version in `.claude-plugin/plugin.json` AND `marketplace.json` ‚Üí add `## [x.y.z] ‚Äî YYYY-MM-DD` entry to `CHANGELOG.md` (`### Added`/`### Fixed`/`### Changed`) ‚Üí commit with explicit author override (`git -c user.name="Paul Ingram" -c user.email="paulingram@users.noreply.github.com"`) ‚Üí annotated tag with same override ‚Üí push main + tag. Consumers update via `/plugin marketplace update <name>` then `/plugin update <name>@<marketplace>` then `/reload-plugins`.
